📐 优化要点对比

🔴 当前问题

  • 使用 about-card 样式，与个人主页布局重复
  • 头像 80px 居中，缺乏视觉冲击
  • 统计数字挤在一行内，无色彩区分
  • 作品卡片不完整（无 hover、无缩略图）
  • 公开歌单使用已删除的旧 CSS 类（已损坏）
  • 好友按钮纯文字，缺乏社交互动感
  • 无在线状态指示
  • 技能展示在简介文本后，无独立区域

🟢 优化方案

  • Hero 头部：渐变背景图 + 大头像 + 在线绿点
  • 统计行：五色卡片式统计（作品/评论/获赞/浏览）
  • 双栏布局：左侧作品列表 + 右侧 sticky 侧边栏
  • 侧边栏：关系按钮组 → 技能标签 → 公开歌单 → 社交链接
  • 作品卡片：2 列网格 + 封面图 + 浏览/点赞数
  • 公开歌单：修复为新的列表式卡片
  • 好友按钮：主按钮 + 私信 + 举报三个层级
  • 在线状态：头像右下角绿点

🛠️ 实施改动清单

1. CSS (`profile.css`) — 新增样式

  • `.profile-hero / .profile-hero-bg / .profile-hero-content` — Hero 头部
  • `.stat-row / .stat-item` — 统计行
  • `.profile-layout` — 双栏 Grid：`1fr 300px`
  • `.up-card / .up-card-img / .up-card-body` — 他人视角作品卡片
  • `.sidebar-card` — 侧边栏卡片复用已有样式
  • `.pl-card / .pl-cover / .pl-info` — 公开歌单列表
  • `.social-link` — 社交链接按钮

2. JS (`profile.js`) — 重写 `renderUserProfile`

  • Hero 头部 → 渐变背景 + 大头像 + 在线绿点 + 操作按钮
  • 统计行 → 五色网格卡片
  • 双栏布局 → 左侧作品 + 右侧侧边栏
  • 公开歌单 → 修复为新 CSS 类名（`.pl-card` 替代 `.music-playlist-card`）
  • 好友按钮 → 添加好友 + 私信 + 举报
