📐 设计对比

▦ 卡片视图（当前）

  • 3 列网格布局
  • 每卡片显示封面大图 + 标题（1行）+ 简介（2行）+ 标签/统计/日期
  • hover 放大封面 + 浮出查看详情遮罩
  • 每屏信息密度低（约 3-6 条）
  • 适合封面精美的作品展示

☰ 列表视图（新）

  • 单列列表布局
  • 每行显示小缩略图 + 标题 + 标签 + 统计 + 日期
  • hover 左侧出现绿色高亮边框
  • 信息密度高（每屏约 8-12 条）
  • 适合快速浏览大量帖子

🛠️ 实施改动清单

1. posts.css — 新增列表视图样式（~100 行）

  • `.post-list` — 列表容器（flex-column）
  • `.post-row` — 单行（flex 布局 + hover 高亮）
  • `.post-row-img / .post-row-body / .post-row-meta / .post-row-stats` — 行内元素
  • `.post-row-title / .post-row-badge / .post-row-tag` — 文字/徽章

2. postList.js — 添加视图切换逻辑

  • 新增 `_viewMode` 属性（'grid' / 'list'）
  • `renderPostList()` 头部添加切换按钮 HTML
  • `_createPostCard()` 保持卡片模式
  • 新增 `_createPostRow()` 生成列表行 HTML
  • 新增视图切换事件 + localStorage 持久化偏好

3. 响应式 — 列表视图在移动端自动适配
