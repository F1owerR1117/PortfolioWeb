# 更新日志

## 2026-05-19 (下午 · 架构优化)

### 重大变更
- **数据库驱动替换**: `sql.js` → `better-sqlite3` (原生 C 扩展, WAL 实时写盘, 零数据丢失)
- **包管理**: 移除 sql.js, 新增 better-sqlite3 12.x
- **db/init.js 重写**: 从 230 行减至 100 行, 去掉内存序列化/定时保存/exit 处理器

### 架构优化
- **LevelService 独立**: `addXP()` 从 `db/init.js` 抽离到 `services/LevelService.js`
- **Migrations 版本化**: `schema_version` 存储在 `settings` 表, 20 个迁移块仅执行未运行过的, 幂等容错
- **动态 require 全面清理**: 30 个文件中 45+ 处函数体内 `require('../db/init')` 全部提至文件顶部
- **新增中间件**: `requireAuthorOrAdmin(table, idParam)` — 消除 posts PUT/DELETE 重复权限检查

### 前端优化
- **音乐界面重构**: `musicLibrary.js` 拆为 8 个独立函数, 编辑/删除/换封面改为局部 DOM 更新
- **播放器优化**: `MusicPlayer._renderPlayer()` 首次 innerHTML, 后续只更新具体元素
- **操作按钮改进**: 歌曲行 4 个挤在一起的按钮 → 1 个 `⋮` 下拉菜单
- **播放次数图标**: ▶ → 🎧
- **系统概览仪表盘**: 完整重写 `admin-stats.js`, 4 分区展示, 今日新增数据, 后端 10 次查询 → 2 次 GROUP BY

### 安全加固
- **SESSION_SECRET 强制**: 去掉硬编码默认值, 未设置环境变量时拒绝启动
- **Session 清理**: XP/level/points 不再写入 session (过期数据源), 前端从 API 实时查询
- **Cookie secure 标记**: 新增 `secure: false` (本地) 配置项, 部署 HTTPS 时改为 true

### 文档
- **新增 `CONVENTIONS.md`**: 11 类编码规范 + 全部模块清单 + 每次 prompt 引用指南
- **新增 `/audit` 技能**: 5 类自动代码审计 (动态 require / Route DB 访问 / XSS / N+1 / CONVENTIONS 合规)
- **文档同步更新**: `index.md`, `architecture.md`, `user-guide.md`, `api-reference.md` 全部更新

- **文字对比度修复**: `--text-secondary` 从 `0.4` → `0.55`，`--text-light` 从 `0.25` → `0.4`，提升可读性
- **评论区风格优化**: 楼中楼 hover 向右滑动、卡片式评论布局（后还原）、粘性侧栏（目录随滚动可见）

- **导航栏**: 简化主题切换图标显示

### 修复
- **sql.js 内存数据库风险**: 文档增加说明。每次写入后 2 秒自动保存，重启电脑可能丢失最后 2 秒数据。uploads/ 文件可能因开发中频繁 kill 进程导致 0 字节残留
- **管理员通知显示异常**: 过滤 `admin_report`/`admin_application` 从普通通知页，未读数不计入铃铛
- **主题切换按钮重复**: 修复导航栏存在两个 `theme-toggle` 元素导致的点击无效
- `checkZoneAccess` 不再拦截未认证用户的求职区 GET 请求，改为前端 UI 提示

## 2026-05-16

### 新增
- **个人主页仪表盘**: `#/profile` 从简单资料卡升级为仪表盘，包含统计面板（作品数、评论数、获赞数、注册天数）、最近作品网格、最近通知列表
- **侧边菜单入口**: 新增「👤 我的主页」菜单项，位于侧边菜单首位
- **用户统计 API**: `GET /api/users/:id/stats` 返回聚合统计数据
- **他人视角切换**: 个人主页可切换到他人视角查看自己的公开页面

### 修复
- **路由竞态**: `renderMyProfile` 拆分为同步骨架 + 异步数据加载（`setTimeout` 延迟），避免 `_handlePath` 中 `updateNav` 等同步操作与异步渲染的时序冲突
- **他人视角返回失效**: 他人视角按钮从直接调用 `renderUserProfile()` 改为 `Router.navigate('#/users/{id}')`，使 URL hash 真正变化，返回按钮的 hashchange 事件得以触发

### 安全
- **CSRF 保护**: Origin/Referer 头验证
- **限流**: 认证接口 10 次/分钟
- **文件上传**: 禁止 .html/.js 上传
