## 快速开始

### 首次启动

```bash
cp .env.example .env   # 创建环境变量文件
npm install            # 安装依赖
npm start              # 启动服务器
```

访问 `http://localhost:3000`，默认管理员账号:
- 用户名: `admin`
- 密码: `admin123`

**安全提示**: 首次登录后请立即修改默认密码！

### 管理员注册

普通用户可直接注册，管理员注册需提供管理员密钥（在 `.env` 文件中设置 `ADMIN_SECRET`）。

### 密码要求

- 长度: 6-128 个字符
- 必须包含: 至少一个字母和一个数字
- 建议: 使用大小写字母、数字和特殊字符的组合

## 用户系统

### 注册与登录

1. 点击导航栏的「登录」按钮
2. 切换到「注册」标签填写用户名和密码
3. 如需注册管理员账号，勾选「管理员」，并填写管理员密钥

### 个人资料

登录后点击导航栏用户头像或用户名，选择「我的主页」：

- **编辑资料**: 点击「编辑个人信息」，可修改昵称、简介、头像、技能标签和社交链接
- **修改密码**: 在编辑页面底部输入当前密码和新密码
- **他人视角**: 点击「他人视角」查看自己的公开主页

### 头像

支持上传头像，上传时可裁剪为 1:1 方形。支持 jpg/png/gif/webp 格式。

## 帖子系统

### 帖子分类

| 分区 | 权限 | 用途 |
|------|------|------|
| 作品区 (Work) | 仅管理员发布 | 展示正式作品 |
| 聊天区 (Chat) | 所有用户可发布 | 日常交流讨论 |

### 发布帖子

1. 点击导航栏的「发布作品」或「发帖」
2. 填写标题、简介、标签（逗号分隔）
3. 可选封面图片（支持裁剪）
4. 添加内容块: 文本、图片、视频、代码、附件

#### 内容块

每个帖子由多个内容块组成，支持排序（↑↓按钮）和删除：

- **文本块**: 多行文本
- **图片块**: 上传图片或输入图片 URL
- **视频块**: 上传视频或输入视频 URL
- **代码块**: 代码编辑器，自动语法高亮
- **附件块**: 上传附件文件，可设置等级/积分解锁和下载（见附件系统章节）

### 帖子操作

- **预览控制**: 编辑时可为内容块设置「允许普通用户预览」
- **置顶/精华**: 管理员可将帖子置顶或标记为精华
- **锁定**: 管理员锁定帖子后，用户无法添加新评论

### 帖子互动

- **点赞/点踩**: 点击帖子底部的 👍/👎 按钮
- **收藏**: 点击 📖 按钮将帖子收藏到收藏夹
- **浏览计数**: 系统自动统计浏览次数（同一用户仅计一次）

## 附件系统

### 上传附件

在编辑帖子时，添加「📎 附件」内容块，点击「上传附件」选择文件。附件最大 50MB。

### 权限设置

每个附件块可设置以下三个维度：

| 设置项 | 说明 |
|--------|------|
| 最低查看等级 | 用户需达到指定等级才能查看附件详情和下载链接 |
| 积分解锁 | 用户需支付指定积分才能解锁附件查看权限 |
| 积分下载 | 解锁后，用户需再支付指定积分才能获得下载权限 |

### 替换与删除附件

编辑已有附件块：
- 点击 `🔄 替换附件` 可重新上传文件，旧文件自动清理
- 点击文件信息旁的 `✕` 按钮可移除附件（保留内容块）
- 删除整个内容块时，关联的附件文件也会自动清理

### 用户视角

根据权限设置，附件区显示不同状态：

| 状态 | 显示 |
|------|------|
| 等级不足 | 🔒 锁定图标 + "需要 Lv.X 才能查看" |
| 需积分解锁 | 🔓 解锁按钮 + "支付 X 积分解锁" |
| 已解锁/需下载 | ⬇ 下载按钮 + "支付 X 积分下载" |
| 完全可访问 | 直接下载链接 |

## 评论系统

### 发表评论

- 在帖子底部输入框填写内容，点击「发布评论」
- 支持 Ctrl+Enter 快捷发布

### 回复评论

- 点击评论下方的「回复」按钮
- 回复会自动嵌套在原评论下方

### 编辑/删除评论

- 自己的评论可编辑或删除
- 管理员可编辑/删除任何评论
- 删除评论会同时删除其所有子评论

### 评论通知

- 当有人评论你的帖子或回复你的评论时，你会收到通知
- 点击通知会自动跳转到帖子详情并滚动到对应评论位置
- 评论区会高亮显示目标评论

## 好友系统

### 添加好友

1. 访问用户的主页
2. 点击「添加好友」按钮发送请求
3. 对方收到通知后，可在好友页面接受或拒绝

### 好友管理

- 在「好友」页面查看好友列表和在线状态
- 可移除好友
- 好友请求数量会显示在导航栏

### 在线状态

- 在线检测: 5 分钟内活跃的用户显示为在线（绿色圆点）
- 好友列表每 30 秒自动刷新在线状态

## 音乐播放器

### 基本功能

底部固定音乐播放器，支持:
- 播放/暂停 (空格键)
- 上一首/下一首
- 进度拖动
- 音量调节
- 播放模式切换: 顺序播放、循环播放、单曲循环

1. 进入「音乐」页面
2. 上传歌曲（名称、艺术家、封面）

4. 创建和管理播放列表

- 播放列表可设为公开，其他用户可查看和收藏
- 公开播放列表有独立的收藏计数

### 键盘快捷键

| 键 | 功能 |
|----|------|
| 空格 | 播放/暂停 |
| → | 下一首 |
| ← | 上一首 |
| M | 切换静音 |

## 举报系统

### 提交举报

在以下位置可提交举报:
- **帖子详情页**: 点击 🚩 按钮
- **用户主页**: 点击「举报用户」按钮

举报需要填写原因，管理员会收到待处理通知。

### 管理举报（管理员）

管理员可在管理面板的「举报管理」中:
- 查看待处理、已处理、已驳回的举报
- **处理**: 标记为已处理（完成审查）
- **驳回**: 标记为无效举报
- **禁言**: 若被举报者为用户，可直接禁言（支持限时/永久）
- **锁定/删除**: 若被举报者为帖子，可直接锁定或删除

## 求职招聘区

求职招聘区（入口：导航栏 `💼 求职招聘` 或侧边菜单）是一个独立的求职/招聘信息版块。

### 进入条件
- 有足够的等级（与作品区/聊天区共享等级体系）
- 无需审核身份即可浏览帖子

### 发帖条件
- 需要先申请并通过招聘者/求职者身份审核
- 审核通过后，点击工具栏 `✏️ 发布帖子` 发帖

### 筛选功能
- **身份筛选**: 全部 / 招聘者 / 求职者
- **工作类型**: 全部 / 全职 / 兼职 / 实习 / 项目
- **城市筛选**: 按城市筛选
- **薪资筛选**: 不限 / 10K+ / 20K+ / 30K+
- **精华帖**: 点击 `⭐ 精华` 查看加精帖子

### 帖子详情
- 求职招聘帖显示位置、工作方式、薪资范围
- 侧栏显示作者信息、联系方式

### 排序
- ⏰ 最新 / 🔥 最热 / ⭐ 精华

## 身份申请系统

### 用户端
1. 进入求职招聘区 → 点击 `📋 立即申请身份`
2. 输入身份类型（招聘者 / 求职者）
3. 输入申请原因（可选）
4. 提交后等待管理员审核
5. 审核通过后即可发帖

### 管理员端
1. 进入后台 → `📋 申请` 标签页
2. 查看待审核申请列表
3. 点击 ✅ 批准或 ❌ 拒绝
4. 审核结果通过通知推送给用户

### 通知
- 用户提交申请 → 管理员收到 `admin_application` 通知（点击跳转 `#/admin/applications`）
- 管理员审核完成 → 用户收到 `job_approved` / `job_rejected` 通知
- 用户举报 → 管理员收到 `admin_report` 通知（点击跳转 `#/admin/reports`）

## 等级与权限

### 获取经验值 (XP)

| 操作 | XP |
|------|----|
| 发帖 | +20 |
| 评论 | +10 |
| 收到点赞 | +2 |
| 帖子加精 | +50 |
| 收到回复 | +5 |

### 等级分区

不同等级可访问不同分区:
- **作品区**: 基础等级即可访问
- **聊天区**: 较高等级才能访问
- **音乐区**: 特定等级可访问

管理员不受等级限制。

### 查看等级

- 导航栏显示当前等级徽章和 XP
- 可在「我的主页」查看详细等级信息
- 管理面板可配置等级名称、图标和阈值

登录后在桌面宽屏（>1660px）显示左右两侧广告栏：

- 广告栏自动填充侧边空间（宽度 200px ~ 400px 动态适配）

## 管理面板

仅管理员可访问。管理面板包含以下功能:

### 侧边菜单

管理员侧边菜单在标准项目下多出折叠的「⚙️ 后台管理」区域，包含全部管理功能。该区域在导航到管理页面时自动展开。

### 数据概览

显示分区统计（帖子数、评论数）、用户总数、待处理举报数。

### 举报管理

见 [举报系统](#举报系统) 章节。

### 用户管理

| 功能 | 说明 |
|------|------|
| 用户列表 | 分页显示所有用户，支持搜索 |
| 禁言 | 临时禁言（指定小时数）或永久禁言，需先确认原因 |
| 解禁 | 解除禁言状态 |

禁言后的用户:
- 无法发布帖子或评论
- 无法发送好友请求或私信
- 无法点赞、收藏、举报
- 无法上传音乐或修改个人资料
- 登录时显示禁言横幅（含剩余时间）
- 收到禁言通知

### 等级管理

- 配置等级名称、图标、背景图
- 设置各等级所需 XP
- 设置各等级可访问的分区
- 手动调整用户等级和 XP

### 标签管理

- 创建和管理帖子标签
- 标签支持分类筛选

### 广告管理

- 创建/编辑/删除广告
- 上传广告图片（建议宽高比 1:2.62）
- 设置广告位置（左侧/右侧）、排序、跳转链接
- 设置广告展示页面（作品区/聊天区）
- 启用/禁用广告

### 弹窗公告管理

- 创建登录弹窗公告（支持标题、内容、图片、链接）
- 编辑和删除公告
- 启用/禁用公告

### 批量操作

- 批量删除帖子
- 批量删除收藏
- 批量删除音乐/播放列表

## 数据库说明

### better-sqlite3 原生驱动

本项目使用 **better-sqlite3**（Node.js 原生 C 扩展），已替代旧版 sql.js。

**特点：**
- 同步 API，WAL 模式实时写盘
- 每次写入即持久化，零数据丢失风险
- 数据库文件: `database.db` + `-wal` + `-shm`

**迁移系统：**
- `schema_version` 存储在 `settings` 表，追踪数据库版本
- 启动时自动执行未运行的迁移，已运行过的永久跳过
- 新增迁移需在 `db/migrations.js` 末尾递增版本号

## 快照与回滚（开发者）

### 创建快照

修改代码前创建快照以保存当前状态:

```bash
bash scripts/snapshot.sh "修改前的状态"
```

### 回滚

```bash
# 查看所有快照
bash scripts/rollback.sh list

# 回滚到指定版本
bash scripts/rollback.sh <commit-hash>
```

支持硬回滚、软回滚和单文件恢复三种模式。
